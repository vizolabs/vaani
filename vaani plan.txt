```
Voice → Whisper Tiny (75MB INT8) → Native Text
                                         ↓
                                NLLB-200 Distilled (190MB INT8)
                                         ↓
                                    T5-Small Grammar (52MB INT8)
                                         ↓
                                  English Output → WhatsApp IME
```


**Total model footprint: ~320 MB** — downloadable once over WiFi, then fully offline.


### Android Implementation Notes


- **Keyboard**: Extends `InputMethodService` — appears system-wide, not just in WhatsApp
- **Inference**: ONNX Runtime Mobile with XNNPACK delegate for ARM NEON acceleration
- **Memory**: Models memory-mapped (`mmap`), not fully loaded into RAM. Active inference uses ~120MB working memory
- **Background**: Model loading happens once at keyboard enable; cached in memory for instant subsequent use

## New Model Choices (Justified)


### Voice → Text: Whisper Tiny INT8 (42MB)


```
Why Whisper Tiny and not Vosk?
├── Vosk Hindi small: 35MB, ~40MB RAM ✓
│   BUT: No Marathi model exists
│   AND: Hinglish voice accuracy is poor
│
├── Whisper Tiny INT8: 42MB, ~55MB RAM
│   Supports: Hindi ✓ Marathi ✓ Hinglish ✓
│   Trained on 99 languages including mr and hi
│   Handles code-switching (Hindi-English mixing) naturally
│
└── DECISION: Whisper Tiny INT8
    Extra 7MB storage = worth it for Marathi + Hinglish support
```


### Translation: MarianMT Bilingual (NOT NLLB)


```
Why MarianMT instead of NLLB-200?


NLLB-200 Distilled (previous choice):
├── 600M parameters → 190MB INT8
├── Supports 200 languages (you need 2)
├── Peak RAM: ~120MB ← TOO MUCH for 1GB phone
└── Total overkill


MarianMT bilingual (new choice):
├── Helsinki-NLP/opus-mt-hi-en (Hindi → English)
│   75M parameters → ~35MB INT8
```

## The Hinglish Problem (Solved)


Hinglish has three forms your father might use:


```
FORM 1: Devanagari Hinglish (typed on Hindi keyboard)
  Input:  "मैं office जा रहा हूँ"
  ──→ MarianMT hi→en handles directly
  Output: "I am going to the office"
  ✓ Works perfectly


FORM 2: Roman Hinglish (typed in English letters)
  Input:  "Main office ja raha hoon"
  ──→ Transliterate to Devanagari first (3MB rule engine)
  ──→ "मैं office जा रहा हूँ"
  ──→ MarianMT hi→en translates
  Output: "I am going to the office"
  ✓ Works with transliteration step


FORM 3: Hinglish Voice (spoken naturally)
  Input:  🎤 [speaks: "kal meeting hai, mujhe report bhejni hai"]
  ──→ Whisper Tiny transcribes (handles Hindi phonetics)
  ──→ "कल मीटिंग है, मुझे रिपोर्ट भेजनी है"
  ──→ MarianMT hi→en translates
  Output: "There is a meeting tomorrow, I have to send the report"
  ✓ Whisper handles code-switching naturally
```


### Transliteration Engine (3MB)


```
Roman Hinglish → Devanagari mapping:


"Main office ja raha hoon"
  M  → म्    a  → अ    in → िन   → मैं
  (space)
  o  → ऑ    ff → फ़    ice → िस   → ऑफ़िस
  (space)  
  j  → ज     a  → अ                → जा
  ...
  
RESULT: "मैं ऑफ़िस जा रहा हूँ"


This is NOT an ML model. It's a lookup table + rules.
Size: ~3MB (covers 50,000 common words + phonetic rules)
Speed: <1ms
RAM: ~2MB
```

## What Changed From Original Design


| Decision | Before | After (1GB) | Why |
|---|---|---|---|
| Translation model | NLLB-200 (190MB) | MarianMT bilingual (35MB) | 200 languages → 2 languages. 5x smaller. |
| Grammar model | T5-Small ML (65MB) | Rule-based (0.5MB) | WhatsApp messages are short. Rules are enough. |
| Model loading | Keep all in memory | Load one → infer → unload | Can't afford 3 models in RAM simultaneously |
| Transliteration | Not needed | Added (3MB) | Roman Hinglish needs Devanagari conversion |
| Voice model | Same Whisper Tiny | Same | Already the smallest viable option |
| Total storage | 320MB | 128MB | 60% reduction |


**The core insight**: when you constrain to just Hindi/Marathi/Hinglish, you can replace a 600M-parameter multilingual model with two 75M-parameter bilingual models. That single change cuts 155MB of storage and halves the RAM requirement.

private var nativeText = StringBuilder()
private var englishTranslation = ""


override fun onKeyDown(keyCode: Int, event: KeyEvent): Boolean {
    when {
        keyCode == KeyEvent.KEYCODE_DEL -> {
            // Backspace
            if (nativeText.isNotEmpty()) {
                nativeText.deleteCharAt(nativeText.length - 1)
                updatePreview()
                // Also update the input field
                currentInputConnection?.deleteSurroundingText(1, 0)
            }
            return true
        }
        else -> {
            // Regular character
            val char = event.unicodeChar.toChar()
            nativeText.append(char)
            currentInputConnection?.commitText(char.toString(), 1)
            updatePreview() // Trigger translation
            return true
        }
    }
}


private fun updatePreview() {
    // Update preview bar with native text
    nativeTextView.text = nativeText.toString()
    // Trigger async translation
    translationPipeline.translate(nativeText.toString()) { english ->
        englishTranslation = english
        englishTextView.text = english
    }
}

    ic?.setSelection(0, nativeText.length) // Select all

}
